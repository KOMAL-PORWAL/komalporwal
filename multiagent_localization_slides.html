‚Ä¢ Update: Weight by measurement likelihood

‚Ä¢ Resample: Eliminate low-weight particles

üåê Factor Graphs

üí¨ Gaussian Belief Propagation

GBP enables scalable inference by passing Gaussian messages between factors and variables.

Message flow:

‚Ä¢ Variable-to-Factor: Fuse incoming messages

‚Ä¢ Factor-to-Variable: Marginalize information

‚Ä¢ Belief Update: Combine all incoming messages

Experimental Results

‚úÖ Single-Agent Particle Filter

Nonlinear System: Successfully localized with bicycle kinematic model (3 states: x, y, Œ∏)
Bearing Estimation: Most challenging parameter; typical error 1-2¬∞
Tuning Required: Process noise Q and likelihood variance L increased ~10x from true values for stability
Measurement Rate: Frequent updates degraded performance; sparse measurements (every 5-10 steps) improved robustness

‚ö†Ô∏è Multi-Agent Challenges

Without Measurements: Odometry-only localization failed rapidly; particles diverged from true path
With GBP: Improved performance when anchor robot (Superbot) shared measurements at 2 Hz (every 5 timesteps)
Bearing Observability: Only position shared; bearing remained nearly unobservable without direct measurements
Communication Rate Impact: Slower communication allowed uncertainty to grow before corrections

üìä Key Finding

Single-agent particle filter converged reliably with proper tuning, but multi-agent GBP required more development. System shows promise but convergence reliability limited by compounded uncertainties and asynchronous communication. Bearing angle remains the bottleneck parameter.

Conclusions & Future Directions

üéØ Key Achievements

Single-Agent Success: Particle filter successfully localizes with nonlinear dynamics and noisy measurements
GBP Implementation: Built factor graph infrastructure from scratch for multi-agent fusion

üí° Lessons Learned

Nonlinearity Challenge: Bearing estimation is most difficult; requires careful parameter tuning

Implementation Complexity: Custom factor graph development is time-intensive; existing libraries (GTSAM) recommended
Multi-Agent Hurdles: Requires more work for true reliability than pure single-agent case
